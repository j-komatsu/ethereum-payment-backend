## DApp・WalletConnect・CPM/MPM の整理

詳細は `docs/report-05F-frontend.md` に体系化済み。
ここでは「知識」として引き継ぐべき要点のみ記録する。

### DApp の実態

- **本来の定義:** サーバーを持たないブロックチェーンアプリ
- **業界での用法:** ウォレットと通信するウェブアプリ全般
- **このプロジェクト:** Spring Boot バックエンドあり = 「完全な DApp」ではない
- **注意点:** バックエンドがある以上、サービス停止・注文拒否は運営側が行える

### WalletConnect の重要ポイント

- QRコードは **DApp 自身がローカルで生成**する（リレーサーバーから返ってくるのではない）
- v2 では **ChaCha20-Poly1305** による E2E 暗号化
- リレーサーバーは暗号文を中継するだけで中身は読めない
- 「DApp は秘密鍵に触れない」は正しいが「何でも安全に署名してよい」ではない
  → 悪意ある DApp は `approve(攻撃者, 全残高)` への署名を誘導できる

### CPM vs MPM

| | MPM（現在の設計） | CPM（将来対応） |
|---|---|---|
| 主導権 | お店側（senderAddress を注文時に指定） | お客側（QRでアドレスを提示） |
| senderAddress | 注文作成時に確定 | 後から判明（QRスキャン後） |

**このプロジェクトの最終形:** MPM × Pull型（Permit + transferFrom）

### 決済方式の分類

```
Push型: transfer() でユーザーが送る（Phase 4 入金検知型）
Pull型: approve/transferFrom でバックエンドが引き落とす（Phase 5 本命）

低遅延決済: Permit + transferFrom（ブロック確認で数秒かかる。「即時」ではない）
事後決済:   Transfer イベント監視（Phase 4）
自動決済:   Sablier / Scheduled（Phase 6）
```

## セキュリティの重要知見

### JWT 保管場所

| 方法 | XSS耐性 | 判定 |
|---|---|---|
| `localStorage` | 🔴 脆弱 | **禁止** |
| メモリ（React state） | 🟢 | リロードで消える |
| `httpOnly Cookie` | 🟢 | **推奨**（SameSite=Strict + CSRF トークン必須）|

### Permit + transferFrom のセキュリティ評価「中」の理由

- バックエンドが秘密鍵を持つのではなく、**spender ウォレットアドレスに transferFrom 権限が集中する**
- spender ウォレットが侵害された場合、承認額の範囲内で引き落とし可能
- `deadline` を短く設定（推奨: 20〜30分）してリスク低減

### EIP-2612 Permit のフィッシングリスク

- Permit 署名はオンチェーンに記録されない
- フィッシングサイトで署名を騙し取られた場合、通常の `approve` より被害が発覚しにくい
- ウォレットが表示するドメインと署名内容を必ず確認すること

## レビュー漏れの記録と再発防止

### 何が起きたか

PR#12（docs/report-05F-frontend.md）を作成・マージした際、
3エージェント並列レビューを実施しなかった。
ユーザーから「特にレビュー指摘なかった？」と指摘されて発覚。

### 指摘で発覚した問題（ドキュメントPRでも🔴が出た）

- WalletConnect QRコード生成フローの誤り
- EIP-2612 Permit「ガス不要」表記の不正確さ
- `AWAITING_CONSUMER` が未実装なのに確定済みのように記述
- senderAddress nullable が現行実装と矛盾
- Permit セキュリティ評価の根拠誤り
- JWT 保管ポリシーの欠落
- nonce のリプレイ攻撃防止説明の欠落
- フィッシングDApp のリスク説明の欠落

→ **ドキュメントPRでも技術的に重大な誤りが複数あった**

### 再発防止策

1. CLAUDE.md に「PR作成後の必須チェックリスト」を追加済み
2. 「ユーザーが『マージして』と言ってもレビュー前はマージしない」を明記済み
3. **毎セッション終わりに knowledge を更新する**（このファイルがその実践）
