## はじめに — ステーブルコインという「実用的な発明」

ビットコインが「デジタルゴールド」として価値保存に使われる一方、
日常決済には「価格が安定していること」が不可欠だ。

ステーブルコインはその解として生まれた。しかし「1ドル = 1ドル」を維持することは、一見単純に見えて、非常に複雑な設計問題だ。

本レポートでは：
1. なぜステーブルコインが使われるのか（現在の価値）
2. どんなリスクが存在するか（過去の失敗から学ぶ）
3. 規制はどう動いているか（日本・EU・米国）
4. 将来どこへ向かうか（CBDC・クロスボーダー・DeFi）
5. JPYC の立ち位置と日本市場の可能性

## 1. ステーブルコインの現在の規模

まず「すでにどれだけ使われているか」の数字から入る。

```
ステーブルコイン市場の時価総額（2026年時点・概算）:
  USDT:   約 $1,200億
  USDC:   約 $600億
  DAI:    約 $50億
  FDUSD:  約 $30億
  合計:   約 $2,000億以上

比較:
  PayPalの1日あたり決済額: 約 $150億
  USDT の1日あたり取引量: 約 $500億〜1,000億（暗号資産全体の決済量で最大）
```

USDT は既にビットコインの日次取引量を超えている日もある。
これは「実験段階」ではなく、**実際に世界中で使われているインフラ**だ。

```
主な利用用途（現在）:
  1. 取引所間の送金（アービトラージ・両替）
  2. DeFi（Uniswap・Aave 等の流動性提供）
  3. 高インフレ国でのドル建て資産保有
  4. 国際送金の代替（SWIFT より速く安い）
  5. リモートワーカーへの給与支払い
  6. NFT・GameFi の決済
  7. BtoB 決済（特に新興国）
```

## 2. ステーブルコインの設計4類型と各リスク

report-06 で基礎を説明したが、将来性を語るためにもう少し深く整理する。

### Type A: 法定通貨担保型（USDC・USDT・JPYC）

```
仕組み:
  発行体がドル（または円）を銀行に預け
  → 同額のトークンを発行
  → 1:1 の交換を保証

USDC（Circle）の透明性:
  - 毎月第三者監査を公開
  - 準備金: 米国短期国債 + 現金
  - 規制当局との協力的な姿勢

USDT（Tether）の不透明性:
  - 長年「本当に担保があるのか」が議論に
  - 2021年の開示では商業手形等の非流動資産も含まれていた
  - 2024年以降は改善傾向

JPYC（HashPort）:
  - 日本の「前払式支払手段」として規制下
  - 1JPYC = 1円の準備金保証
  - 日本の資金決済法に準拠
```

**主なリスク:** 発行体の破綻・資産凍結・規制による発行停止

### Type B: 暗号資産担保型（DAI）

```
仕組み:
  ETH・wBTC などの暗号資産を過剰担保として預け
  → DAI を発行（150%担保 → 1.5倍の価値の担保でDAIを発行）
  → 価格下落時は自動清算（スマートコントラクト）

強み:
  - 特定の発行体なし（誰も凍結できない）
  - コードで動く（透明・検閲耐性）

弱み:
  - 担保の価格が大幅下落すると清算リスク
  - 資本効率が悪い（150%の担保が必要）
  - 現在 USDC 等も担保に含まれており「完全分散」ではない
```

### Type C: アルゴリズム型（TerraUSD — 崩壊例）

```
仕組み（TerraUSD の場合）:
  - 準備金なし
  - 「1UST = $1相当のLUNA（姉妹トークン）と交換できる」という約束
  - 裁定取引によって1ドルを維持する設計

崩壊（2022年5月）:
  UST の大規模売り → LUNAも暴落 → デスパイラル
  → UST: $1 → $0.0001 以下（事実上ゼロ）
  → 損失: 約 $400億（40兆円相当）
  → 投資家の多数が全財産喪失

学んだこと:
  「アルゴリズムだけで価格を維持する」設計は
  需要が崩れた瞬間に完全崩壊する可能性がある
  法定通貨担保や暗号資産担保の「実質的な裏付け」が重要
```

### Type D: コモディティ担保型（XAUT・PAXG）

```
金（ゴールド）などの現物資産と連動させるタイプ。
価格安定は通貨ほど強くないが、インフレヘッジとして機能。
決済用途より資産保有目的で使われることが多い。
```

## 3. なぜステーブルコインが使われるのか — 利用者視点

### 新興国・高インフレ国での意義

```
アルゼンチンの事例（2023年）:
  年間インフレ率: 211%
  銀行口座のペソは1年で価値が3分の1以下に
  → 多くの市民がドル建てステーブルコインに資産を移す
  → 「デジタルドル」として実質的に活用

トルコ・ナイジェリア・ベネズエラでも同様
  ドルを入手するには法的・手続き的障壁が高い
  → USDTを持つことが現実的なドル保有手段になっている
```

これは「投機目的」ではなく、**通貨価値の崩壊から資産を守る実用的な手段**だ。

### リモートワーカーへの給与支払い

```
課題:
  フィリピン・インドのエンジニアに米国からドルを送金
  → SWIFT: 手数料 $30〜50 + 3〜5日

解決:
  USDC を直接ウォレットに送金
  → 手数料: $1〜2（Ethereum L2）または $0.01（Polygon）
  → 時間: 数分

採用企業:
  Gitcoin・DAO 系の多くのリモートチームが USDC 給与を採用
```

### BtoB 決済

```
中小企業の貿易決済の問題:
  信用状（LC）: 数百ドルの手数料 + 数週間
  SWIFT: 高い手数料 + 遅い
  
USDC/USDT による代替:
  → 条件付き決済（Escrow スマートコントラクト）
  → 商品受取確認 → 自動送金
  → 手数料: $0.1〜1
```

## 4. 規制の動向 — 世界のステーブルコイン法整備

### EU: MiCA（Markets in Crypto-Assets）— 2024年発効

```
MiCA の主要規制内容:
  - ステーブルコインは「電子マネートークン（EMT）」として規制
  - EU内での大規模発行には銀行・電子マネー機関ライセンスが必要
  - 1日の取引量が100万件または2億ユーロを超えると「重要EMT」に分類
  - 準備資産の分離保管・定期開示義務
  - アルゴリズム型ステーブルコインの発行は事実上禁止

影響:
  - USDTはEU規制への対応が遅れ、2024年末に一部取引所で上場廃止
  - USDCはMiCA準拠の方向で積極的に対応
  - EU内での「法的に明確なステーブルコイン市場」形成が進む
```

### 日本: 改正資金決済法（2023年施行）

```
日本の規制の特徴:
  「電子決済手段」として法的に定義
  → 銀行・資金移動業者・信託会社のみ発行可能
  → USDT・USDCは現時点で「外国発行の電子決済手段」として扱い検討中

JPYC の立ち位置:
  現在: 前払式支払手段（コンビニ電子マネーと同じカテゴリ）
       → 払い戻し不可、発行残高 2分の1 以上の保全義務
  将来: 電子決済手段への移行を検討中
       → これにより「円の返還請求権」が法的に保証される

重要なポイント:
  日本は「ステーブルコイン = 前払式支払手段 / 電子決済手段」として
  既存金融規制の枠組みに取り込もうとしている
  → 完全な規制空白地帯ではない = 信頼性が高い
```

### 米国: GENIUS Act（2025年成立への動き）

```
GENIUS Act（Guiding and Establishing National Innovation for US Stablecoins）:
  - ペイメントステーブルコイン発行者に連邦ライセンスを義務付け
  - 1:1の法定通貨準備を要求
  - FDICによる預金保護の対象外であることの明示義務
  - 外国発行ステーブルコインの米国市場アクセスに条件

方向性:
  「ステーブルコインを認め、法的に管理する」
  禁止ではなく規制の枠組みに組み込む
  → USDC（米国法人のCircle）は恩恵を受ける立場
  → Tether（BVI法人）は対応を求められる可能性
```

### 規制のまとめ

```
世界の規制トレンド:
  「禁止」ではなく「枠組みに取り込む」

これが意味すること:
  → ステーブルコインは「消える」ものではなく「インフラ化」する
  → 銀行が参入し始める（JPモルガンの JPMD など）
  → 規制コンプライアンスが競争軸になる
  → 法的に明確な発行体（Circle, JPYC）が優位になる
```

## 5. CBDC との関係 — 共存か、競合か

CBDC（Central Bank Digital Currency：中央銀行デジタル通貨）はよく「ステーブルコインのライバル」として語られるが、実態は複雑だ。

```
CBDC の現状:
  デジタル人民元（e-CNY）: 2022年北京五輪で実証
  バハマ「サンドダラー」: 2020年本格稼働（世界初）
  ECBデジタルユーロ: 調査・設計フェーズ（2026年以降）
  日本：2026年度 CBDC 実証実験継続中

日本銀行の方針:
  「発行を決定してはいないが、準備は進める」
  → 技術的な準備を進めながら、政治・社会的判断を待つ段階
```

### CBDC とステーブルコインの違い

```
CBDC:
  発行者: 中央銀行（国家）
  性質:  法定通貨そのもの
  プログラマビリティ: 限定的（政策上の制約）
  プライバシー: 懸念がある（国家が全取引を把握可能）
  DeFi連携: 難しい（規制・政策制約）

ステーブルコイン（USDC・JPYC）:
  発行者: 民間企業
  性質:  法定通貨に連動するトークン
  プログラマビリティ: 高い（ERC-20・EIP-2612等）
  プライバシー: ある程度（ウォレットアドレスは公開だが氏名は不要）
  DeFi連携: 高い（DeFiの流動性の中心）
```

### 共存シナリオ

```
現実的なシナリオ（専門家の多数見解）:
  CBDC:        政府・公共機関の決済・給付金・税収に使われる
  ステーブルコイン: 民間DeFi・国際決済・スマートコントラクト決済で使われる
  → 領域が異なる → 共存

「CBDC がステーブルコインを全廃する」シナリオは考えにくい理由:
  DeFi は「誰も管理しない」ことに価値がある
  CBDC は「国家が管理する」ため、DeFiの文脈では使いにくい
  ERC-20 互換でないと DeFi のスマートコントラクトが使えない
```

## 6. JPYC の将来性 — 日本円決済の可能性

### JPYC の現在地と課題

```
強み:
  - 日本初の日本円ステーブルコイン（ERC-20）
  - 日本の法規制（前払式支払手段）の範囲内で合法運営
  - HashPort Wallet との連携による UX
  - Polygon 上なので低コスト決済が可能

課題:
  - 流動性：USDT・USDC と比べて圧倒的に小さい
  - 払い戻し不可（前払式）→ 円への換金に制限
  - 電子決済手段への移行でより法的明確性が増す可能性
  - DeFi での採用はまだ限定的
```

### 日本市場でのステーブルコイン決済の可能性

```
市場規模（参考）:
  日本のキャッシュレス決済比率: 約 39%（2023年）
  政府目標: 2025年 40%、将来的に 80%
  現在の主役: PayPay・交通系IC・クレジットカード

ステーブルコインが入り込める隙間:
  1. 海外送金（在日外国人・留学生）
  2. フリーランス・リモートワーク給与
  3. BtoB 貿易決済
  4. NFT・ゲーム内課金
  5. DAO・コミュニティ通貨
  6. 国内 EC でのマイクロペイメント（将来）
```

### 2030年に向けたシナリオ

```
楽観シナリオ:
  改正資金決済法による「電子決済手段」への移行
  → JPYC が「円の返還請求権あり」になる
  → 銀行・金融機関との提携が進む
  → EC サイトでの JPYC 支払いが普及
  → 日本の「デジタル円エコシステム」の中核に

現実的シナリオ:
  CBDC（デジタル円）との住み分けで特定ユースケースに特化
  DeFi・GameFi・NFT 市場での日本円建て決済として確立
  年間取引高 1,000億円規模の到達

課題が残るシナリオ:
  規制の不明確さが継続
  USDC の円建てバージョンなど海外勢に席巻される
  JPYCは国内ニッチ市場に留まる
```

## 7. ステーブルコインが変える未来のユースケース

### ストリーミングペイメント（Sablier・Superfluid）

```
従来:
  給与は「月1回・銀行振込」
  フリーランスは「案件完了後・請求書発行・30日後振込」

ストリーミングペイメント:
  1秒ごとに給与が送られてくる
  → 今この瞬間まで働いた分が即時受け取れる
  → 解雇されても「今日の分まで」は確実に受け取り済み

Sablier (report-08) はこれを Ethereum 上で実現するプロトコル
```

### プログラマブル給付金・助成金

```
従来:
  行政給付金 → 申請書類 → 審査 → 数週間後に振込
  不正受給のリスク・手続きコスト

スマートコントラクト活用:
  条件（収入証明のオンチェーン確認）が満たされたら自動送金
  → 給付の自動化・不正排除
  → 事務コストの大幅削減

実際の試みはまだ限定的だが、CBDC + スマートコントラクトの組み合わせで現実味が増す
```

### クロスボーダー決済の標準化

```
現在（2026年）:
  SWIFT GFID: 国際送金の改善プロトコル（2025年本格稼働）
  ステーブルコイン送金: SWIFT 代替として使われ始め

2030年シナリオ:
  JPYC（円） ← → USDC（ドル） ← → EURC（ユーロ）
  DEX（分散型取引所）で即時両替
  → 「為替手数料ゼロ・数秒で着金」の国際送金
```

## 8. 技術的な進化の方向性

### EIP-4337 アカウント抽象化（Account Abstraction）

```
現在の問題:
  ステーブルコインを持っていても ETH がないとガスが払えない
  → 「JPYC は持っているが ETH がないので送金できない」

EIP-4337 で変わること:
  ガス代をステーブルコインで払えるようになる
  → 「JPYC だけ持っていれば JPYC で決済できる」
  → ウォレットのUXが劇的に改善

Permit（このプロジェクトでやっていること）は
EIP-4337 が普及するまでの「現実的な代替手段」でもある
```

### Layer 2 の成熟とコストの収束

```
2024〜2026年の変化:
  Ethereum のEIP-4844（Proto-Danksharding）導入
  → L2 のガスコストが10〜100倍安くなる
  → Polygon での JPYC 送金: $0.001 以下も現実的に

Ultimate目標（Ethereum の将来):
  Danksharding 完全実装
  → 理論上 Ethereum L2 で 100,000 TPS 以上
  → コスト: ほぼゼロ
```

## 9. 結論 — ステーブルコインは「実験」から「インフラ」へ

```
現在（2026年）の位置づけ:
  ✅ 市場規模: 2,000億ドル超（実用フェーズ）
  ✅ 規制: EU(MiCA)・日本(資金決済法)・米国(GENIUS Act)で法整備進行
  ✅ 企業採用: PayPal・Visa・Mastercard が積極関与
  ✅ 技術: L2・EIP-4844 によりコスト問題は解決軌道

将来（2030年）への方向:
  → グローバルな「ドル決済インフラ」としてのUSDC
  → 「円建てデジタル決済」の選択肢としてのJPYC
  → DeFiの基軸通貨としての継続的な役割
  → CBDCとの共存・補完関係の確立

ステーブルコインが「なくなる」シナリオはほぼない。
問題はどのステーブルコインが生き残るか、
どの規制の枠組みの中で、どんな用途で使われるかだ。

このプロジェクトが JPYC を使う意味はまさにここにある：
「日本円でプログラマブルな決済を実現する」という
まだ誰もやっていない領域への最初の一歩として。
```
