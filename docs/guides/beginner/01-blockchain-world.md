# ブロックチェーンの世界 — 例え話で理解する

> 🌱 初心者向け | 読了時間: 約10分  
> 次のステップ: [RPCとAlchemy](02-rpc-and-alchemy.md)

---

## ブロックチェーンを一言で言うと

**「世界中の人が同じコピーを持つ、書き換え不可能な台帳」**

普通の銀行はサーバーに残高を記録します。  
ブロックチェーンは世界中の何万台ものコンピューターが同じ台帳を持つ方式です。

---

## 例え話① 「町内会の回覧板」

```
普通の銀行:
  [あなた] ─── 送金 ──→ [銀行のサーバー]
                           ↑ここだけが記録を持つ
  → 銀行がハックされたら全消え
  → 銀行が倒産したら終わり

ブロックチェーン:
  [あなた] ─── 送金 ──→ [ノードA]─[ノードB]─[ノードC]
                          [ノードD]─[ノードE]─[ノードF]
                          （世界中に何万台も、全員が同じ台帳を持つ）
  → 一部が消えても他が持っている
  → 全員のコピーを同時に改ざんしない限り書き換えられない
```

**回覧板で考えると:**  
町内会の回覧板を全員がコピーして保管しているイメージです。  
1人が「昨日100万円もらった」と書き直しても、他の全員の記録と照合されてバレます。

---

## 例え話② Ethereum は「世界共有の自動販売機」

ブロックチェーン自体は「台帳（記録するだけ）」です。  
Ethereum はそれに加えて「**プログラムも実行できる台帳**」です。

```
普通の自販機:
  100円を入れる → コーラが出る（機械が決まった動作をする）

Ethereum:
  ETHを送る → スマートコントラクトが実行される（コードが決まった動作をする）
```

ポイントは「**誰も止められない**」こと。  
世界中の何万台ものコンピューターで動いているので、  
1台が壊れても・誰かが止めようとしても、動き続けます。

---

## ウォレット = 「デジタルの印鑑 + 口座番号」

| ブロックチェーン用語 | 現実世界の例え | 特徴 |
|---|---|---|
| **秘密鍵（Private Key）** | 印鑑 | 持っている人が本人と証明できる |
| **アドレス（Address）** | 口座番号 | 公開していい（送金先として教えられる）|
| **ウォレット（Wallet）** | 印鑑と通帳のセット | MetaMask などのアプリ |

```
秘密鍵（64文字の秘密): 0xabc...def
    ↓ 数学的な一方向変換（逆算不可能）
アドレス（42文字): 0x1234...5678（これが「口座番号」）
```

> ⚠️ **重要**: 印鑑を紛失したら？  
> 秘密鍵を紛失すると資産が永久に取り出せなくなります。  
> 銀行と違い「再発行」の窓口は存在しません。

---

## トランザクション = 「公証人に認証してもらった書留郵便」

普通のメール: コピーして改ざんできる  
ブロックチェーンのトランザクション（TX）:

```
{
  "from":  "0xAlice...",     ← 送り主（秘密鍵の署名で本人証明済み）
  "to":    "0xBob...",       ← 宛先
  "data":  "transfer 100 JPYC",  ← 命令内容
  "signature": "0x3a4b..."   ← 「これは私が出した命令」という電子署名
}
```

一度ブロックに含まれたら:
- 送った事実は消えない
- 内容は改ざんできない
- 世界中のノードに永久記録される

---

## スマートコントラクト = 「自動実行される契約書」

| 方式 | 内容 |
|---|---|
| 普通の契約 | 「AがBに100万円払ったら、BはAに家を渡す」→ 破ったら裁判所へ |
| スマートコントラクト | 同じ内容をコードで書いてチェーンにデプロイ → 条件が満たされたら**自動的に**実行 |

**JPYC（日本円ステーブルコイン）もスマートコントラクト:**

```
JPYC コントラクトの内部イメージ:

  全アカウントの残高マップ:
    0xAlice = 10,000 JPYC
    0xBob   = 5,000 JPYC
    ...

  transfer(宛先, 金額) が呼ばれると:
    自分の残高 -= 金額
    宛先の残高 += 金額
    「送金した」というイベントを記録
```

## まとめ: 用語早見表

| 用語 | 例え | 一言説明 |
|---|---|---|
| ブロックチェーン | 世界共有の回覧板 | 書き換え不可能な分散台帳 |
| Ethereum | 世界共有の自動販売機 | プログラム実行できるブロックチェーン |
| 秘密鍵 | 印鑑 | これを持つ人が所有者と証明できる |
| アドレス | 口座番号 | 公開していい識別子（42文字）|
| トランザクション | 書留郵便 | 署名済み・永久記録される命令 |
| スマートコントラクト | 自動実行される契約書 | チェーン上のプログラム |
| ノード | 台帳を持つ参加者 | 世界中に何万台も存在 |
