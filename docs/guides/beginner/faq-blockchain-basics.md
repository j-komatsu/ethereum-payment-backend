## Q1. Alchemy って何をしてくれる会社？

**「ブロックチェーンへの窓口（ノード）を代わりに運用してくれる会社」**

```
コンビニ宅配便に例えると:

  あなた     = 荷物を出す人     = あなたのアプリ
  コンビニ   = 受付窓口         = Alchemy
  ヤマト運輸 = 届けて記録する人 = バリデーター

  コンビニは荷物を預かって渡すだけ。届けるのはヤマト。
  Alchemy はリクエストを受けて転送するだけ。記録するのはバリデーター。
```

| Alchemy がやること | Alchemy がやらないこと |
|---|---|
| ✅ ブロックチェーンのデータを読む | ❌ ブロックに記録する |
| ✅ トランザクションをバリデーターに転送 | ❌ トランザクションを承認する |
| ✅ ノード（PC）を24時間運用する | ❌ 残高を書き換える |

## Q2. 法人でも Alchemy を使っていいの？

**はい。法人でも広く使われています。**

```
法人の選択パターン:

  スタートアップ〜中小  →  Alchemy（有料プラン）
  中堅〜大手           →  Alchemy + Infura（2社併用で冗長化）
  超大規模             →  自前ノード + Alchemy バックアップ
```

自前ノードが必要になるのは:
- 月数十億リクエスト超
- データを社外に出せないコンプライアンス要件がある場合

## Q3. RPC って何？

**「離れた場所にあるコンピューターの機能を呼び出す仕組み」**

```
銀行の電話照会に例えると:

  あなた 「残高を教えてください」  ← リクエスト
    ↓ 電話（HTTP）
  銀行  「100万円です」            ← レスポンス

  あなたのアプリ 「このアドレスの残高は？」  ← JSON-RPC リクエスト
    ↓ HTTP
  Alchemy のノード 「1000 JPYC です」        ← JSON-RPC レスポンス
```

実際のデータ:
```json
// 送るもの
{ "method": "eth_getBalance", "params": ["0x1234...", "latest"] }

// 返ってくるもの
{ "result": "0x56BC75E2D63100000" }
```

## Q4. ノードって何？

**「ブロックチェーンの全記録を持つPC」**

```
図書館に例えると:

  ブロックチェーン = 世界共通の百科事典
  ノード           = その百科事典の完全コピーを持つ図書館
  バリデーター     = 新しいページを書き加える権限がある図書館
  Alchemy         = 「代わりに図書館で調べてくるよ」サービス
```

```
ノードの種類:

  ノード（数万台）
  ├── ただのノード ← 記録を持つだけ（Alchemy が運用するのはこれ）
  │
  └── バリデーター ← 記録を持つ + 書き込める（32 ETH が追加で必要）
```

---

## Q5. バリデーターって何？

**「取引を検証してブロックに記録する参加者」**

```
不動産登記所の審査員に例えると:

  ① 売主と買主が契約書を持ってくる     = ユーザーがTXを送信
  ② 審査員が「正当か」確認する         = バリデーターが検証
  ③ 問題なければ登記簿に記録           = ブロックに記録
  ④ 所有権移転が公的に認められる       = 送金が確定
```

## Q6. 1バリデーター = 1台のPC？

**いいえ。1台のPCで複数のバリデーターを動かせます。**

```
「バリデーター」= PCではなく「32 ETH の担保1口」のこと

  個人:     1台のPC × 1口（32 ETH）     = バリデーター 1つ
  お金持ち: 1台のPC × 10口（320 ETH）   = バリデーター 10個
  企業:     数台のPC × 数万口            = バリデーター数万個
```

## Q7. バリデーターしかチェーンに記録できないの？

**はい。ただしトランザクションを「依頼すること」は誰でもできます。**

```
  あなた（誰でもOK）             バリデーター（選ばれた人だけ）
      │                              │
      │ ① 送金命令を作って署名        │
      │ ② ネットワークに送信         │
      │ ──────────────────────────→  │
      │                              │ ③ 検証「署名OK？残高足りる？」
      │                              │ ④ ブロックに記録
      │                              │ ⑤ 他のバリデーターが承認
      │            ⑥ 確定！          │
      │ ←────────────────────────── │
```

## Q8. 32 ETH 預ければ自由にチェーンに書き込める？

**いいえ。正しいトランザクションしか記録できません。**

```
  バリデーターにできること:
    ✅ 正しいTXをブロックに入れる（本来の仕事）
    ✅ TXの順番を選ぶ（チップが高いものを先に）

  バリデーターにできないこと:
    ❌ 他人の残高を書き換える  → 署名がないから無効 → 他の99万台が拒否
    ❌ 存在しないTXを捏造する  → 署名が偽造不可能
    ❌ 不正なブロックを通す    → 2/3以上の賛成が必要 → 1人では無理
```

不正をしたら → 他のバリデーター全員が拒否 + 担保没収（Slashing）

## Q9. バリデーターのメリットは？

**報酬がもらえる。**

```
32 ETH（≈1,200万円）をステークした場合:

  ステーキング報酬: 年利 約3.5%
  → 32 ETH × 3.5% = 年間 約1.12 ETH（≈42万円）
  → 月 約3.5万円

  + チップ収入（ユーザーが「早く処理して」と払う）
  + MEV（トランザクション順序の最適化で得る利益）
```

| 投資先 | 年利 |
|---|---|
| 銀行の定期預金 | 0.01〜0.1% |
| 米国債 | 約4% |
| **ETH ステーキング** | **約3〜4%** |

## Q10. バリデーターはやめられる？ETH は戻る？

**やめられるし、ETH も戻ります。ただし即座には戻りません。**

```
  ① 「やめます」宣言（Exit リクエスト）
      ↓
  ② 待機期間（数時間〜数週間）
      ↓  ← 不正して逃げるのを防ぐための安全装置
  ③ 引き出し待ち（約27時間）
      ↓
  ④ 32 ETH + 報酬が戻ってくる
```

## Q11. バリデーターは個人？会社？

**両方いるが、大半は会社。**

```
  Ethereum バリデーター（約100万台）の内訳:

  ████████████████░░░░░░░░░░░░░░░  Lido等（≈30%）
  ██████████████░░░░░░░░░░░░░░░░░  取引所（≈25%）
  ████████░░░░░░░░░░░░░░░░░░░░░░░  ステーキング企業（≈15%）
  ████░░░░░░░░░░░░░░░░░░░░░░░░░░░  個人ソロステーカー（≈7%）
  ███████████░░░░░░░░░░░░░░░░░░░░  その他（≈23%）
```

個人が少ない理由:
- 32 ETH（≈1,200万円）を出せる個人は少ない
- 24時間サーバー運用が大変
- → だから Lido のような「みんなでお金を出し合う」サービスが人気

## Q12. バリデーターにはノード運用費も必要？

**はい。担保（ETH）+ ノード運用費の両方が必要です。**

```
  バリデーターの総コスト:

  ┌────────────────────────────────┐
  │ ① 担保: 32 ETH（≈1,200万円）   │ ← やめたら戻ってくる
  │                                │
  │ ② ノード運用（月額）           │ ← 毎月かかる
  │    サーバー: 0〜3万円           │
  │    電気代:   2,000〜5,000円     │
  │    回線:     5,000円程度        │
  │    合計: 月 約1万円〜           │
  └────────────────────────────────┘

  報酬: 月 約3.5万円 − 運用費 約1万円 = 月 約2.5万円のプラス
  （ETH価格が変わらない前提）
```

## Q13. Solana など他のチェーンも同じ仕組み？

**「コードが管理する」根本思想は同じ。詳細は違います。**

```
  分散性が高い ←──────────────────────→ 速度が速い

    Ethereum          Polygon          Solana       BNB Chain
    100万台           100台           1,500台        21台
    12秒/block        2秒/block       0.4秒/block   3秒/block
    最も安全          速い             超高速        Binance管理
```

| チェーン | バリデーター数 | 速さ | 特徴 |
|---|---|---|---|
| Ethereum | ≈100万 | 12秒 | 最も分散・安全 |
| Solana | ≈1,500 | 0.4秒 | 超高速だがハイスペックPC必要 |
| Polygon | ≈100 | 2秒 | Ethereumに依存した安全性 |
| BNB Chain | 21 | 3秒 | 実質 Binance が管理 |

## Q14. Alchemy を使う理由（まとめ）

```
自分でノードを立てる場合:
  ストレージ 2TB + 24時間稼働 + 月14〜42万円

Alchemy を使う場合:
  APIキーを発行するだけ + 月0円（無料枠で十分）
```

| | 自前ノード | Alchemy |
|---|---|---|
| 初期費用 | 数万〜数十万円 | 0円 |
| 月額 | 1.5〜4万円（個人）/ 14〜42万円（法人）| 0円（無料枠）|
| 設定時間 | 数日 | 5分 |
| 法人利用 | ✅ | ✅（エンタープライズプランあり）|
